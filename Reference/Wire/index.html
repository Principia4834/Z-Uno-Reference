Wire Library

Implements I2C on Z-Uno pins 9–16. Default is 9 for SCL and 10 for SDA. Other pin range can be selected using ZUNO_REMAP_SPINS()
  • begin()
  • requestFrom()
  • beginTransmission()
  • endTransmission()
  • write()
  • available()
  • read()
  • enableTS()
 Z-Uno can only be I2C master. Default pins can be changed during I2C initialization, but they should be in fast mode pin range: 
{literal}#define SCL_PIN 11
#define SDA_PIN 12

I2CDriver alternative_I2C(SCL_PIN, SDA_PIN); 

void setup() {
    // …
    Wire.bindDriver(&alternative_I2C); // This instructs to use other pins than default
    // …
}{/literal}

Wire Library

This library allows you to communicate with I2C devices. Z-Uno boards have the following wire interfaces:
  • Wire — i2c on SCL(9) and SDA(10) pins
  • Wire1 — i2c on SCL1(23) and SDA1(25) pins
 Кроме пинов по умолчанию можно использовать и другие пины, устанавливаеться это с помощью метода begin()

Обратите внимание, что Wire использует 7 bit адресса(от 0 до 127 - 0 и 127 имеют особое значение). Обратите внимание, что Wire использует буфер размером в 32 байта, если его не достаточно то можете в begin() - указать свой. Обратите внимание, что Wire покуда может работать только в режиме master - возможность работы в режиме slave покуда в разработке. Обратите внимание, что Wire начинает постоянно использовать в режиме slave для приема один DMA канал от begin() до end()
Так же один DMA канал используеться во время передачи данных и во время приема в случае мастера. All Wire have the following methods supported:
  • available()
  • begin()
  • beginTransmission()
  • end()
  • endTransmission()
  • read()
  • requestFrom()
  • setClock()
  • setWireTimeout()
  • write()
